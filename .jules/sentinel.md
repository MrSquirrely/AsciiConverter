## 2024-05-18 - Insecure Temporary Files and Path Traversal
**Vulnerability:** Found predictable temporary file vulnerabilities where temporary file names were hardcoded like `ascii_player_temp_audio.mp3` or `temp_render.mp4` within predictable temporary directories (`Path.GetTempPath()` and local app paths). Also discovered a path traversal vulnerability where user input for project names was blindly concatenated into file paths when seeking playback.
**Learning:** Hardcoded temporary files in a shared or accessible location can be easily guessed, leaving the system open to CWE-377 (Insecure Temporary File) vulnerabilities. Applications are especially susceptible when utilizing 3rd party tools that require passing files via the file system (e.g. ffmpeg or VLC media elements in WPF). Also, relying on UI input directly for file references in playback routines poses path traversal risks if not sanitized correctly.
**Prevention:** Always ensure temporary files are generated using randomized IDs like `Guid.NewGuid():N`. Also, ensure proper file cleanup in `finally` blocks or dispose patterns. For any user input defining local resources, use the same validation and sanitization as used during creation (e.g. stripping `Path.GetInvalidFileNameChars()`).
